Element with class selector

In this case, we can do it as follow in the declaration: p.myclass = {}. No space between the first and second selector. In this case, it means that this style will only be applied to the "p" selector with specifies "myclass"

this is an example as this paragraph has the style of "myclass" but previous do not.

Child selector

In this case, we define the style of a selector that is a child of other selectors. The syntax used is as follows: parent > child {} (i.e. article > p {} - every p that is a direct child of article);

This is an example of child selector. This paragraph is a direct child of an article selector.

This paragraph is NOT a direct child but subchild.

However, this paragraph is not a child and that is why its style is not affected.

Descendent selector

This is how we apply to any child, independently wether it is a direct child or not. The syntax is as follows: parent child {} (i.e. article p {} - every p that is a descendent child of article)

Child selector is applied to this paragraph.

Descendent selector is applied to this "span".
However, this "span" is not a child of article and style is not applied.

Not limited to element selectors

There is also a way that this is not limited to a particular selector. Examples of it are:

  • .myclass p {} (every "p" that is inside (at any level) an element with class ="myclass")
  • article > .myclass {} (every "element" with class="myclass" that is a direct child of article element
